cml文件结构

cml文件

文件组成

chameleon中的组件是以.cml为后缀的单文件形式，其中包括四部分，分别为模板、逻辑、样式与配置信息。例如：

<template>
  <view>
    <view></view>
  </view>
</template>
<script>
export default {

}
</script>
<style scoped>
</style>
<script cml-type="json">
{
  "base":{
    "usingComponents": {
    }
  },
  "wx": {
    "navigationBarTitleText": "index",
     "backgroundTextStyle": "dark",
     "backgroundColor": "#E2E2E2",
     "component": true
  }
  "web": {
  },
  "weex": {
  }
}

1 template

<template></<template>标签中书写组件的视图层结构，chameleon自定义了一套标签语言，结合基础组件、事件系统，可以构建出页面的结构。注意最外层需要是单组件，而不能是多个组件。

  • 模板语法选择 chameleon支持两套模板语法，分别为cml和vue，可以通过在template上设置lang属性进行选择使用某一个语法，一个模板中不可同时使用两种语法。默认的lang为cml

使用vue语法

<template lang="vue">
</template>

使用cml语法

<template lang="cml">
</template>

或者不需要设置lang

<template >
</template>

2 script

<script></script>编写组件逻辑层响应页面操作的代码。标签中书写组件的类，chameleon自定义组件的生命周期，数据管理等，参见逻辑层。 cml文件中不允许使用某个端特定的方法或接口，若有使用，构建时会检查报错。现有支持大量基础API，提供业务调用，@didi/chameleon-api, 以模块的方式引入。 如果你想要使用某个端特定的接口，请从业务出发使用接口多态差异化实现接口。

3 style

<style></style>标签中书写组件的样式，即为chameleon中的CMSS style中书写样式，必须有scoped属性，否则在web端样式会有问题。

4 json

<script cml-type="json"></script>标签中书写组件的配置信息，具体含义参见组件配置

组件的引用

在组件配置一节介绍了组件引用相关字段的使用，如果是组件设置json配置的wx.component为true，如果是引用组件需要把组件的路径设置到usingComponents字段。

注意: 不能在.cml文件中通过import或者require的方式引入.cml文件。
