网络请求

网络请求

目前提供cml.get cml.post方法，更多方法正在支持中。

cml.get

cml.get(OBJECT), 发送get请求，OBJECT参数说明：

参数名 类型 必填 默认值 说明
url String 是 网络请求地址，如果项目中配置了apiPrefix并且setting中的apiPrefix为true，则添加配置的前缀
data Object 否 要传的参数，拼接在请求的url中
header Object 否 设置http请求的header
resDataType String 否 json 设置response的数据类型, 为json时, 会尝试对返回值进行JSON.parse()
setting Object 否 {apiPrefix: true, jsonp: true | false} 自定义了设置，apiPrefix为是否添加chameleon.config.js中设置的apiPrefix; jsonp 为 true 时会发起一个 jsonp 请求
示例代码
cml.get({
  url: 'https://cnodejs.org/api/v1/topics'
}).then(res => {
  cml.showToast({
    message: JSON.stringify(res),
    duration: 2000
  })
}, err => {
  cml.showToast({
    message: JSON.stringify(err),
    duration: 2000
  })
})

cml.post

cml.post(OBJECT),发送post请求，OBJECT参数说明：

参数名 类型 必填 默认值 说明
url String 是 网络请求地址，如果项目中配置了apiPrefix并且setting中的apiPrefix为true，则添加配置的前缀
data Object 否 要传的参数，放到http请求的body中，根据传入的contentType进行类型转换
header Object 否 设置http请求的header
contentType String 否 form 取值：form或json，决定body中data的格式，对应header中content-type为application/x-www-form-urlencoded或application/json
resDataType String 否 json 设置response的数据类型, 为json时, 会尝试对返回值进行JSON.parse()
setting Object 否 {apiPrefix: true} 自定义了设置，apiPrefix为是否添加chameleon.config.js中设置的apiPrefix
示例代码
cml.post({
  url: 'https://cnodejs.org/api/v1/topics',
  data: {
    a: 1
  }
}).then(res => {
  cml.showToast({
    message: JSON.stringify(res),
    duration: 2000
  })
}, err => {
  cml.showToast({
    message: JSON.stringify(err),
    duration: 2000
  })
})

cml.request

cml.request(OBJECT)，OBJECT参数说明：

参数名 类型 必填 默认值 说明
url String 是 网络请求地址，如果项目中配置了apiPrefix并且setting中的apiPrefix为true，则添加配置的前缀
data Object 否 要传的参数，放到http请求的body中，根据传入的contentType进行类型转换
method String 否 GET 若cml.get()/cml.post()无法满足需求,如需使用DELETE/PUT时,可调用此方法
header Object 否 设置http请求的header
contentType String 否 form 取值：form或json，决定body中data的格式，对应header中content-type为application/x-www-form-urlencoded或application/json
resDataType String 否 json 设置response的数据类型, 为json时, 会尝试对返回值进行JSON.parse()
setting Object 否 {apiPrefix: true} 自定义了设置，apiPrefix为是否添加chameleon.config.js中设置的apiPrefix
示例代码
cml.request({
  url: 'https://cnodejs.org/api/v1/topics',
  data: {
    a: 1
  },
  method: 'PUT'
}).then(res => {
  cml.showToast({
    message: JSON.stringify(res),
    duration: 2000
  })
}, err => {
  cml.showToast({
    message: JSON.stringify(err),
    duration: 2000
  })
})
